# λ-1グランプリ

型なしラムダ計算を複数の言語（処理系）で実装・比較する、発表用の実コード置き場。詳細は `docs/proposal.md` を参照。

## Language

**言語 (Language)**:
ラムダ式（関数抽象・適用）を記述する対象となるプログラミング言語そのもの。仕様・文法のレイヤを指す。
_Avoid_: 処理系（実装のことは別語で呼ぶ）

**処理系 (Implementation)**:
ある言語のデモを実際にビルド・実行する具体的なコンパイラ／インタプリタ。1言語に複数ありうる（例: Standard ML には MLton / Poly/ML / SML/NJ）。本 repo は原則として1言語につき1処理系を選ぶ。
_Avoid_: ランタイム, toolchain（より広義）

**デモ (Demo)**:
1言語につき1つ用意する、自己完結した実行可能なラムダ計算の実装単位。聴衆は自分の使う言語のデモだけを単独で読み・動かせる（言語間の再現に依存しない）。デモは原則としてトランスレータが DSL から生成する。
_Avoid_: サンプル, 例

## DSL とトランスレータ

**DSL**:
型なしラムダ計算を記述するための本 repo 独自のソース言語。名前付き定義（`I = λx.x`）と表明（`assert`）からなり、各言語のデモの単一の出処となる。
_Avoid_: スクリプト, 設定

**トランスレータ (Translator)**:
DSL を各言語のネイティブな関数式（クロージャ）へ直接変換し、[[CONTEXT|デモ]]を生成するツール。SKI 等の中間表現は経由しない。
_Avoid_: コンパイラ（狭義の機械語生成と紛れるため避ける）, インタプリタ

**万能型 (Universal type)**:
型なしラムダの全ての値を、静的型付き言語上で表す単一の反射型 `D ≅ D → D`。値は全てチャーチエンコードされた関数なので union を要しない。動的型付き言語では素のクロージャがそのまま値となり万能型は不要。
_Avoid_: タグ付き union, ランタイム型

**プレリュード (Prelude)**:
生成コードが依存する、言語ごとに手書きする境界ヘルパー群。`encode*`（`encodeInt`/`encodeAscii`/`encodeList`/`encodeAssoc` …。host リテラルをチャーチ表現へ）と `decode*`（その逆。言語間で等価判定を揃えるため結果は**文字列表現**を返す）の族、`myassert`（組込み assert が無い言語の代替）、および[[CONTEXT|万能型]]の定義を含む。
_Avoid_: ライブラリ, ランタイム

**境界 (Boundary)**:
ラムダ世界の値と、ホスト言語の native な値（整数・真偽値など）が相互変換される地点。DSL 上ではホストヘルパー呼び出しを `name[…]` と表記し、λ適用の並置と構文的に区別する。
_Avoid_: インタフェース, ブリッジ
