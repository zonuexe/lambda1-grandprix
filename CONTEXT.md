# λ-1グランプリ

型なしラムダ計算を複数の言語（処理系）で実装・比較する、発表用の実コード置き場。詳細は `docs/proposal.md` を参照。

## Language

**言語 (Language)**:
ラムダ式（関数抽象・適用）を記述する対象となるプログラミング言語そのもの。仕様・文法のレイヤを指す。
_Avoid_: 処理系（実装のことは別語で呼ぶ）

**処理系 (Implementation)**:
ある言語のデモを実際にビルド・実行する具体的なコンパイラ／インタプリタ。1言語に複数ありうる（例: Standard ML には MLton / Poly/ML / SML/NJ）。本 repo は原則として1言語につき1処理系を選ぶ。
_Avoid_: ランタイム, toolchain（より広義）

**デモ (Demo)**:
1言語につき1つ用意する、自己完結した実行可能なラムダ計算の実装単位。聴衆は自分の使う言語のデモだけを単独で読み・動かせる（言語間の再現に依存しない）。
_Avoid_: サンプル, 例
