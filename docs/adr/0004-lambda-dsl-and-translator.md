# ラムダ計算 DSL と各言語トランスレータの設計

複数言語でのラムダ式比較（[[CONTEXT]] の「デモ」）を自動生成するため、型なしラムダ計算の DSL（名前付き定義＋`assert`）を各言語のネイティブなクロージャへ**直接翻訳**するトランスレータを Rust で実装する。ラムダ世界の値は**純チャーチエンコード**（全て関数）とし、静的型付き言語では再帰的なタグ付き union の**万能型** `D = Fun (D→D) | Num Int | Str String …` で表現する（ラムダ世界は `Fun` のみ、地の構築子は encode/decode の境界のみ）。

## 主要な決定

- **直接方式のみ**。SKI 中間表現や bracket abstraction は使わない。S/K/I 等のコンビネータは DSL 上の λ 表記ライブラリ定義（`S = λx.λy.λz.x z (y z)` 等）として、他の定義と同様に翻訳する。
- **純チャーチ（ラムダ世界の値は全て関数）**。整数などのデータも全て関数として表現し、host 値への変換は[[CONTEXT|境界]]でのみ行う。ただし decode で「観測」するには地の型が必要で、反射型 `D=D→D` のみでは値が全て関数となり観測できない。そのため型付き言語の[[CONTEXT|万能型]]は `Fun (D→D) | Num Int | Str String …` のタグ付き union とし、地の構築子は境界のみに現れる。動的型付き言語はこの union を untyped 値として暗黙に得る。
- **表面構文**: λ適用＝並置（左結合）、グルーピング＝`( )`、[[CONTEXT|境界]]（[[CONTEXT|プレリュード]]呼び出し）＝`name[…]`。この区別は型付き言語で「λ適用＝万能型の `apply` 経由／ホスト呼び出し＝native 呼び出し」に対応する意味論的に重要な区別であり、単なる見た目ではない。
- **境界ヘルパーは `encode*`/`decode*` の族**: `encodeInt`/`decodeInt`・`encodeAscii`/`decodeAscii`・`encodeList`/`decodeList`・`encodeAssoc`/`decodeAssoc`（Assoc は Lisp の alist 相当のペアのリスト）…と拡張する。`decode*` は言語間で等価判定を揃えるため**文字列表現**を返し、`assert` は文字列等価で比較する。`myassert` は native assert が無い言語の代替（あればそれを使う）。変数名は `_` prefix でマングリングし予約語衝突を避ける。
- **文法の意味論**: 定義は既出の名前のみ参照可（前方参照・暗黙再帰なし。再帰は Y コンビネータで明示）。ラムダ世界に素の host リテラルは置かず、host 値は必ず `encode*[…]` で入り `decode*[…]` で出る（[[CONTEXT|境界]]を常に明示）。
- **実装は Rust 単一クレート**。Lazy K インタプリタ（[[0003-lazy-k-in-rust]]）とパーサ／AST を共有し「λ-1 ツールチェーン」に統合する。
- **バックエンドは T3 ハイブリッド**: 規則的な言語は `LanguageSpec` データ駆動テンプレートで、構造が厄介な言語（Java の関数型インタフェース、C++ の `std::function` ラップ、Go の再帰関数型、Pascal の文構造）のみ `Backend` trait で override する。各言語の[[CONTEXT|プレリュード]]は手書きソースをリポジトリに置き、生成コードをその後ろに連結する。

## Considered Options

- **変換方式**: 直接（採用）／ SKI 経由（bracket abstraction）／両対応。トークの主目的が各言語の native λ 構文の比較のため直接を採用し、SKI は不採用。
- **値表現**: 純チャーチ（採用。ラムダ世界は全て関数）／ host プリミティブをラムダ世界にも流す方式。チャーチはトーク主題（データのエンコード）に直結するため採用。なお型付き言語の万能型は観測のためタグ付き union になるが、地の構築子は境界に限局されラムダ世界の純度は保たれる（当初「反射型のみで足る」と考えたが観測不能のため訂正）。
- **実装言語**: Rust（採用）／ Haskell ／ OCaml。純粋な記述性は Haskell が上だが、Lazy K との統合・単一実装言語・依存最小を優先し Rust。
- **バックエンド構造**: 言語ごとの trait モジュール／宣言的テンプレート／ハイブリッド（採用）。

## Consequences

- 評価戦略は各ホスト言語のものがそのまま表面化する（大半は strict、Haskell のみ遅延）。Y コンビネータが strict 下で発散し Z コンビネータを要する、といった差異はそのまま言語比較の材料になる。
- 全ての値が関数（純チャーチ）ゆえ、λ抽象は常に単引数の入れ子（`λx y.M ≡ λx.λy.M` は糖衣）に翻訳される。多引数クロージャを持つ言語でも入れ子の単引数クロージャで揃え、各言語のカリー化の書き味自体を比較対象にする。
- 残る細部（`λ` と `\` の別、host リテラル記法＝int/文字列、`assert` の `==` トークン）は実装時に詰める。
- `decodeList`/`decodeAssoc`（および対応する encode）は**子要素をどう表現するかが未解決のため実装保留**とする（そもそも不要な可能性もある）。v1 は `encodeInt`/`decodeInt`・`decodeBool` など単純な族に限定する。
